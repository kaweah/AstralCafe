Astral Café

Astronomy components for Java

 

Watch the Astral Applet at work.

(it will take a moment to load and initialize)

This is a demonstration of an applet interacting with servlets, all using the same code base. The applet either preloads star data and calculates projections on its own, or receives projection data from a servlet each time the user requests a view.

The applet can also be initialized with quite a different appearance.

For more information, please see AstralCafe.pdf.

System Requirements

Astral Café is being developed using Java 2 and therefore requires Java 2.

It may or may not operate on Java 1.1.

 

Download the standalone version

AstralCafe.jar

Other files needed to operate the standalone version

log4j-1.2.8.jar
stars3k.dat
links.dat
AstralCafe.properties
log4j.lcf

To launch, execute this command:

java -classpath AstralCafe.jar:log4j-1.2.8.jar AstralCafe AstralCafe.properties

Release Notes


version 1.0

Functionality implemented

  • Stars displayed in color.
  • Use scroll bars to change zenith (or part of sky in view).
  • Draw a box to view a portion of the current view.
  • Switch constellations on/off.
  • Use menus to set right ascension, declination, and width of view.
  • Use a popup menu to set brightness of view.
  • Star name display.
  • Monochrome display modes.


Functionality not yet ported from NightSky to Java

  • All time-related functionality; automatic scrolling.
  • Planets, Moon, and Sun.
  • Messier objects.
  • Status Bar.


Functionality in Night Sky, but not planned for Astral Applet

  • Twilight and Magnitude density display modes.
  • View from Star navigation mode.


Bugs:

  • XOR-based drawing (used in response to mouse drags) does not work cleanly using the J2SE 1.4.1 runtime environment and Safari 1.0.3 (Java 1.3.1 or 1.4.1 Plug-in), but it works fine within IE 5.2/Mac, Netscape 7.1, and OmniWeb 4.2.1. These latter three environments also provide crisper graphics.
  • Dragging to reframe can cause the projection to be reduced to a point on rare occasions. This may be more likely within Netscape 7.1.
